Richard Feynman Digital Twin: Design & Architecture Document1. Project OverviewThis repository contains a Digital Twin of the Nobel Prize-winning physicist Richard Feynman, developed for the AIMS DTU Summer Project 2026. The agent goes beyond standard chatbot responses by emulating Feynman’s distinct teaching style—specifically his insistence on first-principles reasoning and physical analogies—while maintaining deep technical accuracy and stateful memory across sessions.2. Scientist Selection & Persona DesignSubject: Richard FeynmanRationale: Feynman’s corpus (The Feynman Lectures on Physics) provides a dense, highly technical RAG database, ideal for testing the twin's technical accuracy in advanced physics and electrical engineering. His distinct "Feynman Technique" (avoiding jargon, using analogies, intellectual honesty) provides a highly testable persona.Implementation: The persona is strictly enforced via a highly engineered System Prompt that sits inside a LangChain ChatPromptTemplate. It instructs the model to refuse answering questions it cannot reason through ("cargo cult science") and to use conversational, slightly irreverent language.3. Core ArchitectureThe system is orchestrated using LangChain and utilizes Gemini 2.5 Flash as the core inference engine to ensure standardized evaluation.A. The RAG Pipeline (Knowledge Grounding)To ensure the twin cites real ideas instead of hallucinating, we implemented a robust RAG pipeline.Data Source Pivot: We initially attempted to scrape the Caltech HTML archives and parse PDFs. However, standard PDF extractors corrupted the mathematical equations and LaTeX formatting, which would severely degrade the technical accuracy of the RAG pipeline. We pivoted to a pre-cleaned text dataset hosted on Hugging Face (enesxgrahovac/the-feynman-lectures-on-physics).Chunking Strategy: We used a RecursiveCharacterTextSplitter with a strict hierarchy (paragraphs $\rightarrow$ sentences) and a generous overlap (200 tokens) to ensure Feynman's extended analogies were not cut in half.Hybrid Search: We built an EnsembleRetriever combining a dense vector database (ChromaDB with Google's text-embedding-004) for conceptual queries, and a sparse keyword index (BM25) for exact terminology matches.B. Dual-Layer Memory SubsystemTo fulfill the requirement of supporting multi-turn conversations while remembering previous context, the twin uses a dual-layer memory architecture:Short-Term (Session) Memory: Managed via LangChain's ConversationBufferWindowMemory. It feeds a rolling window of recent interactions into the context window, allowing the agent to resolve pronouns and follow immediate follow-up questions without exceeding token limits.Long-Term (Persistent) Memory: Managed via SQLChatMessageHistory. Conversations are persistently written to a local SQLite database (feynman_memory.db) and tagged with unique session_ids. This allows the agent to recall interactions across completely separate sessions, even if the application is restarted.4. Design Trade-offs & DecisionsLocal Vector DB vs. Cloud: We opted for ChromaDB running locally (./feynman_twin_db). This ensures the code is completely reproducible and can be cloned and run instantly by the evaluators without setting up external Pinecone/Weaviate accounts.Ensemble Retrieval over Naive RAG: Standard semantic search often fails on highly specific mathematical constants. Using Reciprocal Rank Fusion (RRF) to blend BM25 with ChromaDB ensures maximum retrieval quality for highly technical physics queries.
